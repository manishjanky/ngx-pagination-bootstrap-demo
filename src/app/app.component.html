ngx-pagination-bootstrap

A Bootstrap(4) based pagination module for Angular(2+) applications.

1.Installation

        //using npm
         npm install ngx-pagination-bootstrap

        //using yarn
         yarn add ngx-pagination-bootstrap
         yarn install

        //Install all the dependencies
        npm install bootstrap@4.0.0-alpha.6 --save-dev
  • Right now bootstrap@4.0.0-alpha.6 is supported will add support for other versions soon
  • include bootstrap scripts and styles in you angular-cli.json to be build of your build process.
  • or include bootstrap css in your index.html
        //in angular-cli.json
         "styles": [
            "../node_modules/bootstrap/dist/css/bootstrap.min.css"
          ]
            
         //in your index.html 
        <link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/bootstrap/4.0.0-alpha.6/css/bootstrap.min.css" 
        integrity="sha384-rwoIResjU2yc3z8GV/NPeZWAv56rSmLldC3R/AZzGRnGxQQKnKkoFVhFQhNUwEyJ" crossorigin="anonymous">

2.Using with webpack and tsc builds/ angular-cli builds

  • import PaginationModule from ngx-pagination-bootstrap
  • add PaginationModule to the imports of your NgModule
        
          import {{'{'}} PaginationModule {{'}'}} from 'ngx-pagination-bootstrap'
        
          @NgModule({{'{'}}
            imports: [
              ...,
          
              PaginationModule
            ],
            ...
          })
          class YourModule {{'{'}} ... {{'}'}}
  • use <ng-pagination></ng-pagination> in your templates to add pagination in your view like below
  • a tag in you html will look like below with all the config
          <ng-pagination [position]="'left'" [pageSize]="10" [itemsCount]="data.length" 
          [data]="data" (getPageData)="getPageData($event)"></ng-pagination>

3.Config

Input
  • pageSize: number - Size of a page i.e number of items to be displayed in one page.
  • itemsCount: number - total no of itmes that needs ro be paged/total no of records.
  • position: string - position of the pagination respect to parent contrainer of your pagination component ['left','center','right'].
  • data: Array - Optional | the data needs to be paged default is null. If pagination is handled by api no need of data.
Output
  • getPageData: EventEmitter - emmited when user navigates to a page. Caputure this and update you view data. The events contains the following info 
        {{'{'}}
            event: event: Event,
            pageNo: pageNo: number,
            pageSize:pageSize: number,
            data: null| Array,
        {{'}'}}
  • event: Event - the triggered event
  • pageNo: number - current page number
  • pageSize: number - current page size. Items per page.
  • data: Array| null - data is the array of records that belongs to this page. Will be null if data was not passed and you can fetch data from your api or the way you want it to handle
